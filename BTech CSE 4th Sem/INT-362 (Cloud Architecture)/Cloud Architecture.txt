Clone

A. Linked Clone
A copy of the guest OS is created, and both the Clone and the Original Guest OS will be using the same resources.
*(Requires less space)

B. Full Clone
A copy of the guest OS is created, and the Clone and the Original Guest OS will be completely independent (means they will be having their own independent resources).

Export to OVF (Open virtualization format)

It is used for creating a file (OVF) which stores the state of the system (files and folders) which you can share to your friends so that they can have the same files and folders which you had in your guest OS (use Open a virtual machine on VMware dashboard).

** There are 3 network adapters
	1. NAT
	-> The host system (VMware's NAT service) behaves like a router, means first the packet from the guest OS is sent to the VMware's NAT service which use the 	host IP to access the internet.
	2. Bridge 
	-> Virtual Machine acts as a separate device (means a new ip will be assigned to the virtual machine using DHCP). 
	3. Host-Only
	-> Host-Only adapter creates a private virtual network which allows communication between the virtual machine and the host only (No internet access by 	default).

VMnet8 → Used for NAT → Internet access (via host)
VMnet1 → Used for Host-Only → Host ↔ VM communication only

Practical 4
----------------------------------------------------------
Ping (default gateway of host wifi) when in bridge mode.
Ping (host ip address / a website) when in NAT mode.
Ping (host ip address) when in Host-Only.

1. Bridged Mode
Ping: Default gateway of host Wi-Fi
Reason: VM is on the same network as the host and uses the same router

2. NAT Mode
Ping: Host IP address or any website (e.g., google.com)
Reason: VM accesses the internet through the host’s IP using VMware NAT service

3. Host-Only Mode
Ping: Host IP address
Reason: Only Host ↔ VM communication is allowed; no internet access
